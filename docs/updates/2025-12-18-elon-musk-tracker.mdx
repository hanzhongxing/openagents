# Demo: Elon Musk News Tracker

A demo that continuously collects news about Elon Musk from various RSS sources and broadcasts them via the **feed mod**. Once running, other agents (including Claude via MCP) can connect and query the feed to answer questions like:

> "What did Elon Musk do in the last 24 hours?"

<Banner type="important">
**Important:** Please stop any running networks before starting this demo.
</Banner>

## What You'll Learn

- Setting up a network with the feed mod for immutable post storage
- Creating a Python-based WorkerAgent for continuous data collection
- Fetching news from multiple RSS/API sources (Google News, Reddit, Hacker News)
- Connecting Claude Desktop or other MCP clients to query your feed

## Connecting Claude Desktop

### Configure MCP Client

**macOS:** Edit `~/Library/Application Support/Claude/claude_desktop_config.json`

**Linux:** Edit `~/.config/claude/claude_desktop_config.json`

**Windows:** Edit `%APPDATA%\Claude\claude_desktop_config.json`

<Banner type="info">
**Note:** Claude Desktop's "Add custom connector" UI requires HTTPS URLs. For local development with HTTP, use this config file method with `npx` to bridge the connection.
</Banner>

```json
{
  "mcpServers": {
    "elon-news-tracker": {
      "command": "npx",
      "args": [
        "mcp-remote",
        "http://localhost:8700/mcp"
      ]
    }
  }
}
```

Restart Claude Desktop after saving the configuration.

### Example Queries

Once connected, try these prompts with Claude:

| Query | What Claude Does |
|-------|------------------|
| "What did Elon Musk do today?" | Calls `get_recent_feed_posts` then summarizes |

## Network Configuration

### network.yaml Reference

```yaml
network:
  name: "ElonMuskTracker"
  mode: "centralized"
  node_id: "elon-tracker-1"

  transports:
    - type: "http"
      config:
        port: 8700
        serve_studio: true   # Web UI at http://localhost:8700
        serve_mcp: true      # MCP at http://localhost:8700/mcp
    - type: "grpc"
      config:
        port: 8600

  mods:
    - name: "openagents.mods.workspace.feed"
      enabled: true
      config:
        max_title_length: 200
        max_content_length: 50000
        enable_search: true
        categories:
          - "news"
          - "elon-musk"
          - "tesla"
          - "spacex"
          - "x-twitter"
          - "neuralink"
```

## Agent Deep Dive

### The WorkerAgent Pattern

The news collector extends `WorkerAgent` with a continuous collection loop:

```python
from openagents.agents.worker_agent import WorkerAgent

class NewsCollectorAgent(WorkerAgent):
    default_agent_id = "elon-news-collector"

    def __init__(self, fetch_interval: int = 300, **kwargs):
        super().__init__(**kwargs)
        self.fetch_interval = fetch_interval
        self.posted_hashes = set()  # Deduplication

    async def on_startup(self):
        self._collection_task = asyncio.create_task(self._collection_loop())

    async def _collection_loop(self):
        while True:
            await self._fetch_and_broadcast_news()
            await asyncio.sleep(self.fetch_interval)
```

### Auto-Categorization

Each news item is automatically tagged based on content:

```python
tags = ["elon-musk", "news"]
if "tesla" in content.lower(): tags.append("tesla")
if "spacex" in content.lower(): tags.append("spacex")
if "twitter" in content.lower() or "x.com" in content.lower():
    tags.append("x-twitter")
if "neuralink" in content.lower(): tags.append("neuralink")
```

## Adding Custom News Sources

Extend `tools/rss_fetcher.py`:

```python
def fetch_custom_rss(url: str, count: int = 10) -> List[Dict]:
    """Fetch from any RSS feed."""
    response = requests.get(url, timeout=15)
    root = ET.fromstring(response.content)

    items = []
    for item in root.findall(".//item")[:count]:
        items.append({
            "title": item.findtext("title", ""),
            "link": item.findtext("link", ""),
            "description": item.findtext("description", ""),
            "published_date": item.findtext("pubDate", ""),
            "feed_source": "custom"
        })
    return items
```

## Troubleshooting

### Network Won't Start

```bash
# Check if port is in use
lsof -i :8700

# Kill existing process
kill -9 $(lsof -t -i:8700)
```

### Agent Can't Connect

```bash
# Verify network is running
curl http://localhost:8700/health
```

### No News Being Collected

1. Check internet connectivity
2. Some RSS sources may be rate-limited
3. Try increasing fetch interval: `--interval 600`

### MCP Connection Failed

```bash
# Test MCP endpoint directly
curl -X POST http://localhost:8700/mcp \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc": "2.0", "method": "tools/list", "id": 1}'
```
