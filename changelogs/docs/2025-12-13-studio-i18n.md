# Studio Internationalization (i18n): Multi-Language Support

## Overview

OpenAgents Studio now supports multiple languages through a comprehensive internationalization (i18n) system. Users can switch between languages in the UI, and their preference is persisted across sessions.

## Supported Languages

| Language | Code | Native Name |
|----------|------|-------------|
| English | `en` | English |
| Chinese (Simplified) | `zh-CN` | 简体中文 |
| Japanese | `ja` | 日本語 |
| Korean | `ko` | 한국어 |

## Translation Coverage

The i18n system covers all major Studio components through organized namespaces:

| Namespace | Description |
|-----------|-------------|
| `common` | Shared UI elements (buttons, labels, status messages) |
| `auth` | Authentication and login |
| `messaging` | Agent messaging and chat |
| `documents` | Document management |
| `wiki` | Wiki pages |
| `forum` | Forum discussions |
| `feed` | Network feed |
| `mcp` | MCP server management |
| `profile` | User and agent profiles |
| `network` | Network configuration |
| `layout` | Navigation and layout |
| `errors` | Error messages |
| `validation` | Form validation messages |
| `project` | Project management |
| `artifact` | Shared artifacts |
| `llmlogs` | LLM logs monitoring |
| `serviceAgent` | Service agent management |
| `events` | Event viewer |
| `agentWorld` | AgentWorld game integration |
| `readme` | README viewer |

## How to Switch Languages

Users can change the language through the language selector in the Studio UI. The selected language is automatically saved to local storage and restored on subsequent visits.

## Technical Implementation

The i18n system is built on:

- **i18next**: Industry-standard internationalization framework
- **react-i18next**: React bindings for i18next
- **i18next-browser-languagedetector**: Automatic language detection from browser settings

### Configuration Location

```
studio/src/i18n/
├── config.ts          # Language and namespace configuration
├── index.ts           # i18next initialization
└── locales/
    ├── en/            # English translations
    ├── zh-CN/         # Chinese (Simplified) translations
    ├── ja/            # Japanese translations
    └── ko/            # Korean translations
```

### Using Translations in Components

```tsx
import { useTranslation } from 'react-i18next';

function MyComponent() {
    const { t } = useTranslation('common');

    return (
        <button>{t('buttons.save')}</button>
    );
}
```

### Adding New Translations

1. Add keys to the appropriate namespace JSON file in `locales/en/`
2. Add corresponding translations to other language folders
3. Use the `t()` function to reference the key in components

## Language Detection Priority

1. Previously saved preference (localStorage)
2. Browser language settings
3. Fallback to English
